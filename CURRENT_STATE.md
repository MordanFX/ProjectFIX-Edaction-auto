# CURRENT_STATE.md

Текущее состояние проекта на 2026-07-14.

Этот файл нужен, чтобы не восстанавливать контекст из переписки перед каждой задачей.

## Главная продуктовая схема

Есть два отдельных потока:

1. Telegram bot + web panel.
2. Discord homework + web panel.

Оба потока попадают в веб-панель, но ученики, курсы и рабочие сценарии должны оставаться разделёнными.

## Telegram-поток

### Уже работает

- `/start` регистрирует Telegram-пользователя.
- Telegram-ученик может получить доступ к курсу.
- Уроки идут поэтапно.
- Урок может содержать несколько материалов/видео.
- Для уроков без ДЗ показывается, что сдавать ничего не нужно.
- Для уроков с ДЗ ученик может сдать работу.
- Ученик может отправлять текст, файл, фото или видео.
- Куратор может проверять Telegram-ДЗ из Telegram-режима или из веб-панели.
- Куратор может принять работу или отправить на доработку.
- Ответ куратора может содержать текст и вложение.
- Ученик получает результат проверки обратно в Telegram.
- Telegram-курсы и ученики отображаются отдельно от Discord.

### Важно

- Telegram-бот и веб-панель не вызывают друг друга напрямую.
- Бот и панель работают через базу.
- У куратора в Telegram должен быть `telegram_user_id`, привязанный к `staff_users`.

### Что ещё требует улучшения

- Удобная страница управления командой/кураторами.
- Скрывать кураторов из списка Telegram-учеников, если они нажали `/start` только для режима куратора.
- Более понятная выдача курса через Telegram-режим куратора.
- История действий куратора.
- Закрепление работы за конкретным куратором.

## Discord-поток

### Уже работает

- Discord-бот подключён к серверу клиента.
- Slash-команда `/homework` создаёт личное homework-пространство ученика.
- У каждого Discord-ученика своя приватная ветка/пространство.
- Ученики не видят ветки друг друга.
- Discord-ученик может отправить ДЗ на проверку.
- У Discord-сообщения с ДЗ есть кнопка отправки на проверку.
- Есть сценарий вопроса к куратору.
- Вопросы попадают в панель/очередь вопросов.
- Discord-ДЗ попадают в веб-панель.
- Куратор может принять работу или отправить на доработку из веб-панели.
- Ответ куратора может содержать текст и вложение.
- Ответ уходит обратно в приватную Discord-ветку.
- В панели есть история попыток по ДЗ.
- В панели можно открыть исходное сообщение в Discord.
- Фото/изображения ученика можно просматривать из панели.

### Discord-права

Доступ к приватным Discord-веткам задаётся не веб-аккаунтом, а ролями Discord.

Роли, которые обсуждались как staff:

```text
Founder
Main Trader's Department
Support Department
```

Их ID, которые передавались для staff mention:

```text
1324489727155372174
1400407463584338001
1437783918802894951
```

Для доступа к private homework-веткам в Discord нужно настроить права канала/категории:

```text
Просмотр канала
Отправлять сообщения
Читать историю сообщений
Отправлять сообщения в ветках
Просматривать приватные ветки
```

Если куратор не видит ветку, проблема обычно в Discord permissions/roles, а не в веб-панели.

## Веб-панель

- Telegram ученики.
- Discord.
- Выдать ДЗ Discord.
