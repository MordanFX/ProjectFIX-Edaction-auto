# FIX BY MRDN — Дизайн-код (Brand & Design System)

Этот файл описывает фирменный стиль бренда **FIX BY MRDN**. Используй его каждый раз при создании любого визуального ассета (презентация, HTML-инструмент, обложка, карточка, интерфейс). Все правила ниже обязательны к соблюдению — от них нельзя отступать без явного указания.

---

## 1. Общая философия

- Премиальный, «тёмный» tech-стиль: почти чёрный фон, глубина за счёт синего радиального свечения.
- Минимализм и воздух. Много негативного пространства, ничего лишнего.
- Строгая иерархия: один крупный акцентный заголовок на экран, остальное — тихо и структурно.
- Серебро + чёрный — доминирующая гамма. Синий — редкий акцент-заливка, не «раскрашивание».
- Настроение: дорого, технологично, уверенно, без «инфоцыганского» глянца.

## 2. Цветовая палитра

### Основные (фон и база)
| Роль | HEX | Комментарий |
|------|-----|-------------|
| Фон основной (near-black) | `#0D0D0D` | Базовый фон почти всех слайдов |
| Фон глубокий (чёрный) | `#030303` | Затемнённые края, виньетка |
| Фон холодный (сине-чёрный) | `#010510` | Вариант фона с синим подтоном |
| Поверхности / карточки | `#16161A` – `#1B1B1E` | Панели, блоки, чуть светлее фона |

### Акцент (использовать СДЕРЖАННО)
| Роль | HEX | Правило |
|------|-----|---------|
| Синий акцент (brand blue) | `#2074EF` | ТОЛЬКО как заливка (плашки, линии, свечение, активные элементы). **НЕ используется для текста и иконок.** |
| Синий свет (glow) | `#0263F2` → прозрачность | Радиальное свечение фона |

### Текст и нейтрали
| Роль | HEX | Комментарий |
|------|-----|-------------|
| Текст основной / заголовки | `#FFFFFF` | Белый для дисплейных заголовков |
| Текст вторичный (серебро) | `#BCBCBD` | Подписи, body, лейблы |
| Текст приглушённый | `#7A7A7C` | Технические подписи, сноски |
| Разделители / обводки | `#2A2A2E` | Тонкие линии 1px |

> **Правило синего:** синий (`#2074EF`) — это заливка и свет, а не цвет текста/иконок. Если сомневаешься — не крась синим.

## 3. Фон (обязательный приём)

Почти-чёрный фон с **радиальным синим свечением** (vignette + glow):
- База: `#0D0D0D` / `#030303`.
- Радиальный градиент синего свечения (`#2074EF` / `#0263F2`) из смещённой точки (обычно сверху/сбоку), быстро затухающий к краям.
- Тёмная виньетка по краям кадра для глубины.
- Генерируется программно (Python + NumPy + Pillow) как растровая подложка, а не плоская заливка.

Пример логики генерации подложки:
```python
# Радиальное синее свечение на near-black фоне
import numpy as np
from PIL import Image

W, H = 1920, 1080
cx, cy = int(W*0.5), int(H*0.35)     # смещённый центр свечения
yy, xx = np.mgrid[0:H, 0:W]
dist = np.sqrt((xx-cx)**2 + (yy-cy)**2) / (W*0.6)
glow = np.clip(1 - dist, 0, 1) ** 2.2  # мягкое затухание

base = np.array([13, 13, 13])          # #0D0D0D
blue = np.array([32, 116, 239])        # #2074EF
img = base[None,None,:] + glow[...,None]*(blue-base)*0.35
Image.fromarray(img.astype('uint8')).save('bg.png')
```

## 4. Типографика

| Назначение | Шрифт | Начертание |
|------------|-------|------------|
| Дисплейные заголовки | **Montserrat** | ExtraBold (800) |
| Body / подписи / лейблы | **Manrope** | Regular / Medium / SemiBold |

Правила:
- Заголовки — Montserrat ExtraBold, крупно, часто UPPERCASE или с сильным контрастом размеров.
- Тело и подписи — Manrope, спокойно, с увеличенным межстрочным.
- Один доминантный заголовок на экран. Не более 2–3 уровней размера.
- Трекинг у крупных заголовков — слегка расширенный; у body — нейтральный.
- Montserrat в вариативном формате → извлекать статические веса через `fonttools`.

CSS fallback:
```css
--font-display: "Montserrat", system-ui, sans-serif;   /* 800 */
--font-body: "Manrope", system-ui, sans-serif;         /* 400/500/600 */
```

## 5. Логотип / вордмарк

- Вордмарк **FIX BY MRDN**.
- Позиция по умолчанию: **верхний правый угол** ассета.
- Монохром: серебро/белый на тёмном. Синим логотип не красить.
- Размер сдержанный — присутствие, а не доминирование. Держать защитное поле вокруг.
- Файл логотипа в проекте: `Logo_для_чата_4.jpg` (на чёрном фоне).

---

## 6. Композиция и лейаут

- Сетка с крупными полями, контент не прижимается к краям.
- Точка внимания одна: крупный заголовок ИЛИ ключевой график/схема.
- Графики (свечные, схемы аукциона, зоны) — на тёмном фоне, линии тонкие, акценты синим-заливкой.
- Разделители — тонкие 1px линии `#2A2A2E`, не жирные.
- Карточки/панели — чуть светлее фона (`#16161A`–`#1B1B1E`), скругление умеренное, без тяжёлых теней; глубина за счёт свечения фона.
- Иконки — линейные, тонкие, серебро/белые. Не синие.

## 7. Продакшн-стек (как это собирается)

| Инструмент | Назначение |
|------------|------------|
| **PptxGenJS** | Основная генерация презентаций |
| **Node.js** | Рантайм для сборки деков и HTML-инструментов |
| **Python / NumPy / Pillow** | Генерация радиальных виньеток-подложек |
| **LibreOffice headless + pdftoppm** | Конвертация в PDF и растеризация слайдов для QA |
| **fonttools** | Извлечение статических весов Montserrat из вариативного шрифта |
| **Self-contained HTML** | Интерактивные инструменты офлайн: base64-ассеты + CDN Google Fonts как fallback |

Процесс: сборка кодом → экспорт в PDF → растр в JPG для визуального контроля перед выдачей.
