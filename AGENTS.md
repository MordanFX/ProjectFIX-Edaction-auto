# Правила разработки

## Границы проекта

- Следовать этапам из `PROJECT_OVERVIEW.md`; не смешивать будущие этапы с текущим.
- Бот и веб-панель не вызывают друг друга напрямую. Общая точка интеграции — база данных.
- Общие модели и правила предметной области находятся в пакете `course_platform`.
- Telegram Bot API вызывается напрямую через асинхронный `httpx`, без bot-фреймворков.

## Код и данные

- Поддерживаемая версия Python: 3.12+; весь ввод-вывод выполняется асинхронно.
- Изменения схемы базы оформляются миграциями Alembic.
- Секреты читаются только из окружения/`.env`; `.env` не коммитится.
- Идентификаторы Telegram хранятся как `BIGINT`.
- Порядок уроков и попыток сдачи защищается ограничениями базы, а не только кодом.

## Качество

- Новое поведение сопровождается тестами.
- Целевое покрытие — не ниже 70%.
- Перед завершением этапа выполняются `pytest --cov` и `ruff check .`.
- Не добавлять инфраструктуру продакшен-деплоя до отдельного решения по хостингу.

## Непринятые продуктовые решения

До подтверждения клиентом не фиксировать в бизнес-логике:

- открытие следующего урока после сдачи или после зачёта;
- допустимые форматы домашнего задания;
- окончательный способ хранения видео;
- состав ролей и точные права кураторов;
- правила платного и ознакомительного доступа.
